# Frontend Design Fusion — 前端设计融合规范

一套既大胆创意又专业可执行的前端设计指南。让 AI 输出摆脱千篇一律，同时保证设计质量和可用性。

---

## 核心原则

### 1. 非AI风格第一

**这是最高优先级。**

AI 生成内容的通病：

- ❌ Inter + 紫色渐变 + 圆角卡片
- ❌ 左右图文交替布局
- ❌ 三列特性展示
- ❌ 悬浮出现 + 弹性动画

**破解方法：**

- 选择一种极端美学方向并贯彻到底
- 使用意想不到的布局（对角线、重叠、破格网格）
- 字体要有性格，拒绝系统默认
- 色彩要么极克制，要么极大胆，不要中间态

### 2. 浅色优先，暖调为主

**默认使用浅色主题，避免深色：**

- ✅ 背景：米白 #faf9f7、暖灰 #f5f3f0、淡黄 #fefcf8
- ✅ 文字：深棕 #1a1714、炭灰 #2d2a26
- ✅ 强调色：陶土色、橄榄绿、锈红、芥末黄、珊瑚粉

**如需深色元素：**

- 仅用于小面积强调（按钮、标签、图标）
- 避免大面积深色背景

### 3. 大胆美学方向（选一而终）

每个项目选择一种方向，彻底执行：

| 方向            | 特征                               | 适用场景               |
| --------------- | ---------------------------------- | ---------------------- |
| **新粗野主义**  | 粗边框、原始材质、无装饰、系统字体 | 创意机构、作品集       |
| **编辑/杂志风** | 大字号、大量留白、不对称网格       | 内容型网站、博客       |
| **有机/自然**   | 流动形状、大地色系、手绘感         | 生活方式、环保品牌     |
| **复古未来**    | 像素风+渐变、霓虹点缀、网格背景    | 科技、游戏、创意       |
| **软/粉彩**     | 低饱和糖果色、圆角、柔和阴影       | 女性向、儿童、温馨场景 |
| **工业/实用**   | 等宽字体、标签式UI、数据密集       | 工具、仪表盘、开发者   |

## 设计规范

### 字体 (Typography)

**推荐组合（选一套）：**

| 风格     | 标题字体         | 正文字体      | 调性       |
| -------- | ---------------- | ------------- | ---------- |
| 现代人文 | Playfair Display | Source Sans 3 | 优雅编辑   |
| 几何清晰 | Space Grotesk\*  | DM Sans       | 现代友好   |
| 复古质感 | Fraunces         | IBM Plex Sans | 温暖专业   |
| 工业技术 | JetBrains Mono   | Inter\*       | 开发者工具 |
| 东方韵味 | Noto Serif SC    | Noto Sans SC  | 中文内容   |

\*Space Grotesk 和 Inter 已被过度使用，仅在工业/工具场景使用

**规则：**

- 建立字体比例系统（1.25 或 1.414 比例）
- 正文行高 1.6-1.8
- 最大行宽 65ch

### 色彩 (Color)

**使用 OKLCH 色彩空间：**

```css
:root {
  /* 暖调浅色背景 */
  --bg-primary: oklch(0.98 0.01 80); /* 暖白 */
  --bg-secondary: oklch(0.94 0.02 80); /* 米灰 */

  /* 文字 */
  --text-primary: oklch(0.2 0.02 60); /* 深暖棕 */
  --text-secondary: oklch(0.45 0.03 70); /* 中灰 */

  /* 强调色（选一） */
  --accent-terracotta: oklch(0.6 0.15 45); /* 陶土 */
  --accent-olive: oklch(0.65 0.1 120); /* 橄榄 */
  --accent-rust: oklch(0.55 0.18 30); /* 锈红 */
  --accent-mustard: oklch(0.75 0.12 95); /* 芥末 */
  --accent-coral: oklch(0.7 0.15 25); /* 珊瑚 */
}
```

**规则：**

- 中性色永远带暖色调（不要纯灰 #888）
- 强调色只用 1-2 个，不要彩虹
- 彩色背景上文字要够深（对比度 4.5:1 以上）

### 空间 (Spacing)

**基础网格：**

- 基础单位：8px
- 间距尺度：8, 16, 24, 32, 48, 64, 96px

**布局原则：**

- 内容区最大宽度：1280px
- 移动端边距：16-24px
- 板块间距：64-128px
- 不要害怕留白

### 动效 (Motion)

**缓动函数：**

```css
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);
```

**时长规范：**

- 微交互（hover, focus）：150-200ms
- 页面过渡：300-500ms
- 复杂动画：600-800ms

**原则：**

- 一次精心编排的入场动画 > 到处散落的微交互
- 尊重 `prefers-reduced-motion`
- ❌ 禁止弹性/bounce 动画

### 交互 (Interaction)

**Focus 状态：**

```css
:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 2px;
}
```

**Loading 状态：**

- 优先 skeleton，其次 spinner，避免仅用文字

**错误提示：**

- ❌ "输入有误"
- ✅ "邮箱需要包含 @ 符号"

## 执行规则

1. **生成 UI 时自动应用本规范**
2. **浅色暖调优先**，除非用户明确要求深色
3. **选择一种美学方向**并彻底执行
4. **避免所有 AI 常见模式**（紫色渐变、Inter 字体、三列卡片）
5. **收到命令时**：先说明修改计划，再输出代码
