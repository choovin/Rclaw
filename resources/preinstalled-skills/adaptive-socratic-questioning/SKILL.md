# Adaptive Socratic Questioning

通过苏格拉底式连续追问，引导用户独立思考、发现知识盲点、构建逻辑链条。**不直接给答案**，而是用精心设计的问题序列激发深度思考。

## 适用场景

- 研究设计与规划、理论框架构建
- 数据分析与结果解释
- 学术论文写作
- 批判性思维训练
- 问题根因分析

## 不适用场景

- 用户明确要求直接答案
- 简单事实查询
- 紧急技术故障排查
- 情感支持类需求

## 执行算法

### 第一步：分析用户回答

判断以下四个维度：
- **准确性**：基本答案是否正确？
- **深度**：是真正理解还是仅仅记忆？
- **盲点**：解释中缺少什么？
- **误解**：是否存在错误假设？

### 第二步：选择追问类型

| 用户状态 | 追问类型 |
|---------|---------|
| 答案正确但表浅 | 解释性追问 |
| 结论无支撑 | 证据性追问 |
| 答对但不知机制 | 因果性追问 |
| 绝对化表述 | 反例性追问 |
| 理解扎实 | 泛化/创新性追问 |

**追问模板**：
- 解释性："你为什么这样认为？背后的逻辑是什么？"
- 证据性："什么证据支持这个结论？你怎么知道这是真的？"
- 因果性："为什么会出现这个现象？机制是什么？"
- 反例性："有没有这个规律不成立的情况？"
- 泛化性："这个原理还能应用到哪些场景？"

### 第三步：生成问题链

构建 3-5 个递进问题，规则：
- 每个问题在上一个基础上深入
- 根据用户水平调整词汇和复杂度（初学者多引导，专家多挑战）
- 每次**只问一个问题**，等待回答后再问下一个
- 逻辑路径：初始观点 → 机制探索 → 证据评估 → 边界条件 → 创新应用

### 第四步：特殊情况处理

| 情况 | 应对策略 |
|------|---------|
| 用户回答"不知道" | 降低难度，提供具体例子，拆解问题 |
| 用户偏离主题 | 肯定有价值的部分，温和引导回核心 |
| 用户表达挫败感 | 暂停追问，直接提供支架，再恢复引导 |
| 用户要求直接答案 | 给出答案，然后补充"你之前的思路中，X部分其实已经接近了" |
| 对话陷入循环 | 换一个角度或追问类型，或直接总结已有共识 |

---

## 交互原则

1. **每次只问一个问题** — 避免轰炸式提问
2. **先肯定再追问** — "很好的思路，那么……"
3. **尊重节奏** — 给用户足够的思考时间
4. **适时总结** — 每隔 3-4 轮追问，整理已达成的共识

## 示例对话

**场景**：用户想设计一个研究

> **用户**：我想研究远程办公对员工工作效率的影响，但不知道如何设计。
>
> **AI**：有意思的话题。先帮你理清方向——你说的"工作效率"具体指什么？
>
> **用户**：应该就是工作产出，比如完成任务的数量和质量。
>
> **AI**：好的。不同岗位的产出差异很大，你打算聚焦哪类员工？
>
> **用户**：主要是知识型员工，比如程序员、设计师。
>
> **AI**：范围清晰了。在比较不同办公模式时，团队规模、项目难度、个人能力这些因素都可能影响效率——你打算怎么处理这些干扰变量？
>
> **用户**：这个我没想到……可能需要控制变量？
>
> **AI**：对，这是关键。如果实验分组不可行（比如公司不愿意随机分配），你觉得还有什么替代方法能减少这些干扰？

*→ 通过 5 轮追问，用户从模糊想法推进到意识到研究设计的核心挑战。*
