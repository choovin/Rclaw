## 二、需求回顾

### 2.1 品牌化
- 所有对外展示的 UI 文案，全部由 RClaw 改为 RClaw

### 2.2 UI 美化
- 参考 RClaw-resimg/11.png 完成 UI 和交互的改造
- 只做部分调整

### 2.3 文案修改
- "新对话" → "新建对话"
- "模型" → "安全模型"
- "Agents" → "数字员工"
- "频道" → "连接IM"
- "技能" → "技能商店"

### 2.4 数字员工标签页面
- 内置两个子标签："员工市场" 和 "我的员工"
- 员工市场：导入 agency-agents-temp 的 agents，按 OpenClaw 格式转为 MD 文件
- 按不同部门排序，每个员工起中文名字（来自近15年热播电视剧人物姓名）
- 点击"添加"可添加到"我的员工"
- 我的员工：点击员工名字，右侧栏显示概览、频道、技能等信息

### 2.5 其他 UI/UX 优化

每个任务完成后需验证：

## 十一、风险与注意事项

1. **数据兼容性**：现有 agents 数据需要迁移
2. **国际化**：zh/en/ja 三语言需同步修改
3. **性能**：员工市场加载大量数据需分页/懒加载
4. **测试**：全面测试新建对话、添加员工等核心流程
