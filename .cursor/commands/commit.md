# Git 提交（中文说明）

执行一次规范的本地提交：**提交标题与正文须使用中文**描述改动；专有名词、文件路径、API 名称、命令与标识符可保留英文。风格与仓库根目录 `AGENTS.md` 中「Git 提交信息」一节一致。

## 执行步骤

1. **确认范围**：在仓库根目录运行 `git status`；需要时用 `git diff` 与 `git diff --staged` 看清未暂存与已暂存差异。
2. **暂存文件**：对本次要纳入提交的路径执行 `git add`。不要暂存密钥、环境文件、仅本地调试产物或明显误改的文件。
3. **撰写提交说明**：
   - **标题**（首行）：一句概括本次改动，建议动词开头，简洁可读（约 50 字内）。
   - **正文**（可选）：标题下空一行，再分条写动机、主要变更、注意事项或后续跟进。
4. **创建提交**：使用 `git commit`；多段正文可用多次 `-m` 或编辑器多行信息，确保最终展示为「中文描述 + 必要时英文技术词」。
5. **自检**：提交后可用 `git log -1` 确认说明是否准确反映 diff。

若当前没有可提交的有效变更、或用户未指定要包含的文件导致范围不明，先说明情况并询问，不要猜测性 `git add -A` 后强行提交。
